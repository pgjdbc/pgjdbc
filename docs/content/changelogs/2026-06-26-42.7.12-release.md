**Notable changes**

These are the changes most likely to affect you when upgrading from 42.7.11. Three of them change default behaviour; see [Changed](#changed).

* **DDL no longer breaks reused prepared statements.** With the new `flushCacheOnDdl` property (default `true`), the driver re-prepares server statements after `CREATE`/`DROP`/`ALTER`, so a reused statement no longer fails with `cached plan must not change result type`. Previously this needed `autosave=ALWAYS`. Set `flushCacheOnDdl=false` to keep the old behaviour [PR #4067](https://github.com/pgjdbc/pgjdbc/pull/4067).
* **`search_path` changes are detected on PostgreSQL 18+.** PostgreSQL 18 reports every `search_path` change to the client, so a change hidden inside PL/pgSQL or a function now invalidates the prepared-statement cache instead of leaving a stale plan behind [PR #4259](https://github.com/pgjdbc/pgjdbc/pull/4259).
* **Faster batch inserts.** `reWriteBatchedInserts` now merges up to 32768 rows into one multi-values `INSERT` (bounded by the 65535 bind-parameter protocol limit) instead of capping at 128. The new `reWriteBatchedInsertsSize` property lowers that cap when you need to [PR #4207](https://github.com/pgjdbc/pgjdbc/pull/4207).
* **XA recovery works on managed datasources.** Crash-recovery on TomEE, WildFly, and WebSphere Liberty no longer fails with `2nd phase commit must be issued using an idle connection`. As part of this, `setAutoCommit(false)` and `setSavepoint(...)` on an enlisted connection now throw instead of silently doing nothing [PR #4114](https://github.com/pgjdbc/pgjdbc/pull/4114).
* **`getLong`/`getInt`/`getShort`/`getByte` are correct for overlong numbers.** Reading a text-format numeric column whose value overflowed `long` could silently return a wrong number (for example `1234` instead of an error); it now falls back to the slow path and raises the proper `Bad value` error [PR #4163](https://github.com/pgjdbc/pgjdbc/pull/4163).
* **User classes load in Quarkus, OSGi, and application servers.** The new `classLoaderStrategy` property lets the driver fall back to the thread context classloader when loading `socketFactory`, `sslfactory`, and similar user classes [PR #4167](https://github.com/pgjdbc/pgjdbc/pull/4167).
* **SSL works on FIPS-mode JVMs.** Connections no longer fail on FIPS JVMs (Semeru FIPS 140-3, IBM SDK in FIPS mode, BouncyCastle FIPS) that offer no general-purpose `KeyStore` type [PR #4193](https://github.com/pgjdbc/pgjdbc/pull/4193).

### Added

* feat: `reWriteBatchedInserts` now merges up to 32768 rows into one multi-values `INSERT`, bounded by the 65535 bind-parameter limit of the extended protocol, instead of capping at 128. The new `reWriteBatchedInsertsSize` connection property lowers that cap; the default `0` uses the protocol-derived maximum [PR #4207](https://github.com/pgjdbc/pgjdbc/pull/4207).
* feat: `flushCacheOnDdl` connection property (default `true`) invalidates the prepared-statement cache after `CREATE`/`DROP`/`ALTER`, so callers no longer hit `cached plan must not change result type` without opting into `autosave=ALWAYS`. Set `false` for the prior behaviour [PR #4067](https://github.com/pgjdbc/pgjdbc/pull/4067).
* feat: `classLoaderStrategy` connection property controls which classloaders the driver searches when loading a class named by a connection property (for example `socketFactory`). The default `driver-first` now falls back to the thread context classloader, which fixes class loading in non-flat class paths such as Quarkus and OSGi. Use `driver` for the previous driver-only behaviour, or `context-first` to prefer the thread context classloader [PR #4167](https://github.com/pgjdbc/pgjdbc/pull/4167).
* feat: `connectExecutor` and `connectExecutorArg` connection properties customise the `Executor` that runs the connection attempt while `loginTimeout` is in effect. The value is the fully qualified name of a class implementing `java.util.concurrent.Executor`. The default runs the attempt on a daemon thread named `PostgreSQL JDBC driver connection thread`, as before [PR #4165](https://github.com/pgjdbc/pgjdbc/pull/4165).
* feat: `LargeObject.supports64BitOffsets()` and `Fastpath.getServerVersionNum()` let callers pick the 64-bit large-object fastpath (`lo_tell64`, `lo_lseek64`, PostgreSQL 9.3+) by server version instead of calling it and recovering from the failure on older servers [PR #4204](https://github.com/pgjdbc/pgjdbc/pull/4204).
* feat: `BlobInputStream.skip(long)` now seeks with `lo_lseek`/`lo_lseek64` instead of reading and discarding bytes, so skipping a large distance no longer costs a full read [PR #4204](https://github.com/pgjdbc/pgjdbc/pull/4204).
* feat: add `Oid` constants for the geometric array types (`LINE_ARRAY`, `LSEG_ARRAY`, `PATH_ARRAY`, `POLYGON_ARRAY`, `CIRCLE_ARRAY`), `RECORD`, and `RECORD_ARRAY`. `REF_CURSOR`/`REF_CURSOR_ARRAY` are renamed to `REFCURSOR`/`REFCURSOR_ARRAY` to match `pg_type`, with the old names kept as aliases [PR #4220](https://github.com/pgjdbc/pgjdbc/pull/4220).

### Changed

These entries change default behaviour. Review them before upgrading.

* change: `reWriteBatchedInserts` rewrites larger batches by default (up to 32768 rows, was 128). Existing batches that relied on the 128-row split now produce fewer, larger `INSERT` statements. Set `reWriteBatchedInsertsSize` to restore a lower cap [PR #4207](https://github.com/pgjdbc/pgjdbc/pull/4207).
* change: the prepared-statement cache is invalidated after DDL by default (`flushCacheOnDdl=true`). Set `flushCacheOnDdl=false` to keep the prior behaviour, in which a reused statement could fail with `cached plan must not change result type` after `ALTER TABLE` [PR #4067](https://github.com/pgjdbc/pgjdbc/pull/4067).
* change: on an XA-enlisted connection, `setAutoCommit(false)` and `setSavepoint(...)` now throw, where they used to be a silent no-op. `commitPrepared` and rollback-of-prepared return `XAER_RMFAIL` (was `XAER_RMERR`) when the connection arrives in a non-idle transaction state, which transaction managers retry instead of abandoning the prepared transaction [PR #4114](https://github.com/pgjdbc/pgjdbc/pull/4114).
* perf: the connection attempt under `loginTimeout` now runs as a `FutureTask` and is cancelled with `cancel(true)` on timeout, so a slow attempt is interrupted instead of leaking a worker thread. There is no public API change [PR #4120](https://github.com/pgjdbc/pgjdbc/pull/4120).
* perf: defer simple-query flushes until the driver reads the response, so an implicit `BEGIN` and the query that follows it share one network flush [PR #4196](https://github.com/pgjdbc/pgjdbc/pull/4196).

### Fixed

#### Connections, SSL, GSS, and proxies

* fix: auto-detect the SSL key format instead of assuming PEM for a `.key` file, which broke DER-encoded `.key` files with `MalformedInputException`. Detection is by extension (`.p12`/`.pfx`, `.pem`, `.der`) and falls back to scanning the file for a PEM header [PR #3946](https://github.com/pgjdbc/pgjdbc/pull/3946).
* fix: build PKIX trust anchors without a `KeyStore`, so SSL connections work on FIPS-mode JVMs that offer no general-purpose `KeyStore` type [PR #4193](https://github.com/pgjdbc/pgjdbc/pull/4193).
* fix: GSS connections now honour `gssResponseTimeout` rather than `sslResponseTimeout` [PR #4076](https://github.com/pgjdbc/pgjdbc/pull/4076).
* fix: merge the default non-proxy hosts with a system-supplied `socksNonProxyHosts`, so a SOCKS proxy no longer receives an unresolved `127.0.0.1` and fails with `UnknownHostException` [PR #4045](https://github.com/pgjdbc/pgjdbc/pull/4045).
* fix: close the socket when `PgConnection` setup fails after the network connect, instead of leaking the open socket [PR #4161](https://github.com/pgjdbc/pgjdbc/pull/4161).
* fix: reject a null URL in `Driver.acceptsURL` with a clear `NullPointerException` instead of an opaque parsing error [PR #4205](https://github.com/pgjdbc/pgjdbc/pull/4205).

#### Statements, types, and results

* fix: `getLong`, `getInt`, `getShort`, and `getByte` on a text-format numeric column no longer return a silently wrong value when the input overflows `long`; an out-of-range magnitude now falls back to the slow path and raises `Bad value` [PR #4163](https://github.com/pgjdbc/pgjdbc/pull/4163).
* fix: `ResultSet.getBigDecimal(int, int)` now honours the requested scale, rounding with `HALF_EVEN` instead of throwing, and applies it on the binary path too. Negative `numeric` scales (PostgreSQL 15+) are decoded correctly [PR #4211](https://github.com/pgjdbc/pgjdbc/pull/4211).
* fix: `PreparedStatement.toString()` no longer throws for a `bytea` value supplied as text via `PGobject`; hex values are validated and rendered as a `bytea` literal, and escape-format values are quoted and cast [PR #4201](https://github.com/pgjdbc/pgjdbc/pull/4201).
* fix: writing a `java.time` value read with `getObject(int, <type>.class)` straight back through an updatable `ResultSet` (`updateRow()`/`insertRow()`) no longer throws `ClassCastException` [PR #3848](https://github.com/pgjdbc/pgjdbc/pull/3848).
* fix: a comment before `CALL` in a `CallableStatement` no longer hides the native call, so `registerOutParameter` works for `/* comment */ call proc(?, ?)` [PR #4209](https://github.com/pgjdbc/pgjdbc/pull/4209).
* fix: `ResultSet.getCharacterStream` wraps the `String` in a `StringReader` instead of copying it into a `char[]` [PR #4063](https://github.com/pgjdbc/pgjdbc/pull/4063).
* fix: `DatabaseMetaData.getColumns` reports `CHAR_OCTET_LENGTH` only for character and binary columns and `null` for the rest, and declares the column as `INTEGER` [PR #4231](https://github.com/pgjdbc/pgjdbc/pull/4231).
* fix: `DatabaseMetaData.getPrimaryKeys` reads the key columns from `pg_constraint.conkey`, which drops a server-version check and returns the correct keys on PostgreSQL-compatible engines that under-report `pg_index` metadata [PR #4202](https://github.com/pgjdbc/pgjdbc/pull/4202).
* fix: an updatable `ResultSet` over an unqualified table name is classified using only the table visible through `search_path`. When two schemas held a table of the same name with the same primary or unique index name but different key columns, the driver merged both schemas' columns and could wrongly reject the result set as not updatable [PR #4214](https://github.com/pgjdbc/pgjdbc/pull/4214).
* fix: `PGInterval.setSeconds` rejects out-of-range and `NaN` values instead of silently wrapping past about 68 years [PR #4194](https://github.com/pgjdbc/pgjdbc/pull/4194).
* fix: `timestampadd`/`timestampdiff` reject the `SQL_TSI_FRAC_SECOND` interval with an explained error instead of silently producing values off by a factor of 1000 [PR #4229](https://github.com/pgjdbc/pgjdbc/pull/4229).

#### Prepared-statement cache and `search_path`

* fix: detect `search_path` changes from the PostgreSQL 18 `GUC_REPORT` message, so a change made inside PL/pgSQL or a function invalidates the prepared-statement cache [PR #4259](https://github.com/pgjdbc/pgjdbc/pull/4259).
* fix: detect `search_path` changes case-insensitively and handle `RESET search_path` and `RESET ALL`, so an upper-case `SET SEARCH_PATH` no longer leaves stale cache entries [PR #4216](https://github.com/pgjdbc/pgjdbc/pull/4216).

#### Batching and the wire protocol

* fix: `reWriteBatchedInserts` no longer throws `IllegalArgumentException` when batching a parameterless `INSERT` (for example `INSERT INTO t VALUES (1, 2)`) of 256 rows or more [PR #4207](https://github.com/pgjdbc/pgjdbc/pull/4207).
* fix: improve batch grouping when `RETURNING` contains `varchar` or `numeric` columns, whose length lives in the type modifier rather than the type [PR #4014](https://github.com/pgjdbc/pgjdbc/pull/4014).
* fix: restore the pre-describe step for generated-key batches, so large `INSERT ... RETURNING` responses are sized correctly and cannot deadlock the connection [PR #4014](https://github.com/pgjdbc/pgjdbc/pull/4014).
* fix: do not throw `AssertionError` from `BatchResultHandler` when the connection is already closed (for example, the backend terminated mid-batch); a `BatchUpdateException` propagates instead [PR #4187](https://github.com/pgjdbc/pgjdbc/pull/4187).
* fix: add an explicit failure message when a multi-statement is added to a batch, which `addBatch` does not support [PR #4014](https://github.com/pgjdbc/pgjdbc/pull/4014).
* fix: avoid creating a transient `ResultSet` while describing a statement, which could interfere with `Statement.closeOnCompletion()` [PR #4014](https://github.com/pgjdbc/pgjdbc/pull/4014).

#### Large objects

* fix: `BlobInputStream` mark and reset are now relative to the `LargeObject`'s current position, so a stream created from an already-seeked large object resets to the correct byte. A failed seek no longer leaves stale buffered data behind [PR #4204](https://github.com/pgjdbc/pgjdbc/pull/4204).

#### XA and JTA

* fix: `PGXAConnection` no longer saves and restores the caller's JDBC `autoCommit` flag. Every XA-protocol command is sent with `QUERY_SUPPRESS_BEGIN`, so the caller's `autoCommit` is invariant across `XAResource` calls. This fixes the `2nd phase commit must be issued using an idle connection` failure during recovery on TomEE, WildFly, and WebSphere Liberty [PR #4114](https://github.com/pgjdbc/pgjdbc/pull/4114).
* fix: `PGXAConnection.prepare()` mutates XA state only after `PREPARE TRANSACTION` succeeds, so a failed `PREPARE` no longer leaves the driver issuing `ROLLBACK PREPARED` against a transaction the server never saw [PR #4114](https://github.com/pgjdbc/pgjdbc/pull/4114).

#### Driver lifecycle, classloaders, and resource hygiene

* fix: do not null the `contextClassLoader` of shared `ForkJoinPool.commonPool()` workers, which left unrelated tasks running with a `null` classloader [PR #4156](https://github.com/pgjdbc/pgjdbc/pull/4156).
* fix: clear the `ResourceBundle` cache on `Driver.deregister()` so the driver classes can be unloaded once a localised message has been formatted [PR #4237](https://github.com/pgjdbc/pgjdbc/pull/4237).
* fix: keep the `LazyCleaner` task alive across a transient empty queue, so bursty load on a CPU-constrained host no longer accumulates cleanup actions to `OutOfMemoryError` [PR #4038](https://github.com/pgjdbc/pgjdbc/pull/4038).
* fix: delete the temporary file when spooling a stream to disk fails with an `IOException`, instead of leaving it behind [PR #4190](https://github.com/pgjdbc/pgjdbc/pull/4190).
* fix: parse `maxResultBuffer` without a direct dependency on `java.lang.management`, so it works on runtimes that lack it, such as Android ART [PR #4069](https://github.com/pgjdbc/pgjdbc/pull/4069).
* fix: simplify the `Statement#cancel` state machine by dropping the redundant `CANCELLED` state, which removes a spin-forever case when more than one thread observes the cancel completing [PR #1827](https://github.com/pgjdbc/pgjdbc/pull/1827).

#### autosave

* fix: skip the autosave savepoint for `SET LOCAL TRANSACTION` and `SET SESSION TRANSACTION`, which PostgreSQL rejects inside a subtransaction, so these statements work under `autosave=always` [PR #4203](https://github.com/pgjdbc/pgjdbc/pull/4203).

### Release engineering

* fix: the published GitHub release now ships the released `postgresql-<version>.jar` and its detached PGP signature, taken byte-for-byte from the signed build uploaded to Maven Central, instead of a leftover SNAPSHOT jar [PR #3814](https://github.com/pgjdbc/pgjdbc/pull/3814).

* Simplified Chinese: fill in the untranslated messages, carrying forward translations contributed by [Feng Zhihao](https://github.com/Fzhlib) in [PR #2123](https://github.com/pgjdbc/pgjdbc/pull/2123) that never merged [PR #4208](https://github.com/pgjdbc/pgjdbc/pull/4208).
* Russian: revise the catalog, by [Vladimir Sitnikov](https://github.com/vlsi) with a review from [Sergey Mokhov](https://github.com/smokhov) [PR #1280](https://github.com/pgjdbc/pgjdbc/pull/1280).
* Refresh translations across all 15 catalogs after the XA error messages were reworded, and convert the remaining ISO-8859 catalogs (Czech, German, French, Italian, Dutch) to UTF-8, so every catalog in the tree is UTF-8 [PR #4114](https://github.com/pgjdbc/pgjdbc/pull/4114).
